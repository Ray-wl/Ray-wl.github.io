JavaScript数据交互思路的整理

使用JavaScript操作DOM或BOM都只是其主要工作的一部分，在目前越来越多的数据驱动页面的需求来看，JavaScript操作数据在未来的工作中将会占很大的比重。

那么使用JavaScript如何获取数据？如何操作数据？如何对数据进行处理？是近期学习的重点。

对于JavaScript操作数据，之前的学习中已经学习了一些入门的基础知识，在学习了解react和Vue时，关于MVVM模型，这些都是有初步了解，但一方面不深入，另一方面长时间没有复习，所以此次关于JavaScript同数据之间的学习仍然需要从零开始这个自己需要认识到。

关于JavaScript同数据的联系，我初步整理了一个思维导图，图中都是依据自己的理解而表达的，如下：

通过该图可以总结出以下主要学习的知识点：

1. ajax方法

可以通过原生JavaScript或jQuery使用ajax方法访问数据，但是面临两个问题

①. 无法直接访问本地数据

②. 无法直接访问远程数据

换句话说就是仅仅使用ajax什么数据都访问不成，所以引出后面的两个知识点：Node JS和跨域。

2. Node JS（静态文件托管）

使用ajax方法无法直接访问本地数据，必须通过创建本地服务器才能访问，创建服务的方法有很多，如SublineServer、Tomat服务器、node js、window系统自带的IIS功能、XAMPP软件等。

Node js本身就是运行在服务端的JavaScript，是一个基于Chrome JavaScript运行时建立的一个平台。他是基于JavaScript，入门更能简单一些（当然深入学习仍是有难度的），所以选择了node js的方式创建本地服务（托管静态文件）。

3. 跨域

对于跨域最常见的理解是狭义的跨域，是由浏览器同源策略限制的一类请求场景。

至于何为跨域，以及解决跨域的多种方法，我将在学习后用一整篇的博文来总结。
